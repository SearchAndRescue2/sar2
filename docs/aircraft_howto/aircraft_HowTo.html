  4. Set aircraft characteristics

    5. If you don't work at an aircraft factory, the easier way is to look for aircraft real characteristics on the internet. You will have to find the following data (here is an example for the EC145 helicopter), and include them in the description tag as below. In game, these data will be printed on the "Aircraft Details..." screen.

  5. Adjust helicopter Flight Model (FM) values

    6. Helicopter FM values will define how your helicopter will fly and react to player steering. Below is how I went about doing it. There is maybe (certainly?) a better way, but this method seems to give acceptable results...

    1. First, start Sar II, select Free Flight, select a Theater and a location, select the aircraft that you are working on, then click on the [Begin] button, then set units [CTRL+U] to English. Remember this: in english unit, speed will be indicated in knots in cokpit view [F2] and in miles per hour in external view [F3]. Then, finish game ( [ESC], [y] ). It is important to finish game now because new FM values are loaded when you press the [Begin] button. You will have to finish game at each time you want to modify a FM value.

    2. Stall and speed_max are not used for a copter, you can set them to zero. Define min_drag to a value between let's say 0.8 for a modern and very efficient helicopter and 1.5 for an old and/or hugly one. Set overspeed_expected at Max Speed value and overspeed at Never Exceed Speed (formaly VNE, Velocity to Never Exceed).
      # Speed bounds, all in mph
      # 133 knot = 153 mph, 145 knot = 166.9 mph, 150 knot = 172.6 mph.
      #    stall  speed_max  min_drag  overspeed_expected  overspeed
      speed 0.0      0.0       0.95           166.9          172.6

    3. Set fuel value.
      # Fuel capacity and efficiency. For the consumption_rate we just use the equation:
      # consumption_rate = fuel_max_kg / max_endurance_time_in_seconds / avg_throttle_coeff
      # where avg_throttle_coeff is the average throttle value (ie 0.8 for most cases).
      # consumption_rate(kg/sec)  initial_kg  max_kg
      fuel       0.093              694.0     694.0

    4. Set dry mass as this: dry_mass = aircraft standard mass at takeofff - fuel initial_kg . It's a good idea to calculate a second dry_mass for helicopter max weight too, because we will need it later.
      # Dry mass, in kg (EMS equipped, with 2 pilots)
      dry_mass 2450.0
      # EC-145 dry_mass for max weight test:
      #dry_mass 2891.0

    5. Set engine settings.
      To define power, in cockpit view [F2], at 80% throttle/collective, flight forward at a constant altitude (speed is not important at this time), and adjust engine power to obtain a cockpit inclination angle of about* -12°. Head Up Display as a mark at -10°, but not at -12°. It will help you if you add a mark at -12°: I made that with an arrow cutted up in a Post-it® and pasted on the monitor.
      * I found the -12° angle value on the internet and decided to use it as a goal to reach. # Engine settings.
      #  can_pitch?  initial_pitch   power   collective_range
      engine  n            0        39275.0       0.50

    6. Set helicopter climb rate.
      Adjust vertical acceleration responsiveness (helicopter_accelresp "k" coefficient) to reach maximum rate of climb at 100% throttle (collective). More "k" = less climb rate, less "k" = more climb rate. Climb rate is shown in external view [F3]. I read that in real life maximum climb rate is reached only when helicopter flights forward but Sar II is a game, thus let's say that we want to reach it when helicopter moves vertically only.
      # Helicopter acceleration responsiveness. Higher values produce less responsiveness.
      # EC145 max climb rate: 1600ft/min = 26.7ft/s (for max rate of climb at 100% throttle)
      #                   right/left  fwd/bckwd  up/down
      #                        i          j         k
      helicopter_accelresp  4200.0     1455.0     1065.0

    7. Stabilize engine power and helicopter acceleration responsiveness "k" coefficient values.
      Iterate the two points above until both values are stable before going to next point.

    8. Set helicopter horizontal cruise speed.
      Adjust horizontal acceleration responsiveness (helicopter_accelresp "j" coefficient) to reach cruise speed when flying at constant altitude with 80% throttle (collective). More "j" = less speed, less "j" = more speed.
      # Helicopter acceleration responsiveness. Higher values produce less responsiveness.
      # EC145 max climb rate: 1600ft/min = 26.7ft/s (for max rate of climb at 100% throttle)
      #                   right/left  fwd/bckwd  up/down
      #                        i          j         k
      helicopter_accelresp  4200.0     1455.0     1065.0

    9. Set left/right acceleration responsiveness "i" coefficient.
      Copy forward/backward helicopter_accelresp "j" coefficient to left/right "i" coefficient.
      # Helicopter acceleration responsiveness. Higher values produce less responsiveness.
      # EC145 max climb rate: 1600ft/min = 26.7ft/s (for max rate of climb at 100% throttle)
      #                   right/left  fwd/bckwd  up/down
      #                        i          j         k
      helicopter_accelresp  1455.0     1455.0     1065.0

    10. Set attitude_leveling heading, pitch, and bank values.
      Except heading which must be null for an helicopter, I don't have any rule for pitch and bank values... Test and do it as you feel it!
      # Attitude leveling, in degrees per second
      #                   h    p    b
      attitude_leveling  0.0  5.0  10.0

    11. Set dry_mass at maximum mass.
      Set dry mass as this (you should already have calculate it...): dry_mass = aircraft maximum mass at takeofff - fuel initial_kg .
      # Dry mass, in kg (EMS equipped, with 2 pilots)
      #dry_mass 2450.0
      # EC-145 dry_mass for max weight test:
      dry_mass 2891.0

    12. Start game and check if helicopter can takeoff (it should).

    13. Reset dry mass to its original value:
      # Dry mass, in kg (EMS equipped, with 2 pilots)
      dry_mass 2450.0
      # EC-145 dry_mass for max weight test:
      #dry_mass 2891.0

    14. Set collective_range.
      Start game, takeoff, stabilize altitude between 10 to 20 feet Above Ground Level, then suddenly bring throttle/collective to the minimal position: helicopter must crash and explode. If not, increase the collective_range value (max value: 1.0).
      # Engine settings.
      #  can_pitch?  initial_pitch   power   collective_range
      engine  n            0        39275.0       0.50

      15. Your helicopter is now ready to fly!
